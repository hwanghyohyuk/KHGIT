기능
	로그인
	1. 사용자(회원,지점관리자) 의 아이디와 비밀번호를 입력받아 서버로 전송
	2. 서버는 받은 정보를 자신이 가지고있는 파일 시스템(users파일)에서 찾음
	3. 있으면 클라이언트로 True 반환, 없으면 False 반환
	4. 클라이언트에서 받은 데이터를 가지고 로그인 성공 및 실패를 보여줌

	회원가입
	회원
	1. 클라이언트에서 사용자가 사용자데이터를 입력함
	2. 아이디 중복검사 
	- 비밀번호 입력
	- 이메일 입력
	- 연락처 입력
	3. 회원가입 버튼 클릭시 이벤트 발생
	4. 이벤트 발생 내용 
	- 이메일, 연락처가 유효성에 맞게 입력되었는지 확인
	- 완료가 되었다면 다음을 실행, 완료되지 않았다면 다시 입력 받도록 함
	- 입력받은 정보를 user객체로 만들어 서버에 전송
	7. 서버는 받은 정보를 users파일(테이블)에 저장
	- 성공적으로 저장되면 클라이언트에게 완료메시지 전달

	지점관리자
	1. 클라이언트에서 사용자가 사용자데이터를 입력함
	2. 아이디 중복검사 
	- 비밀번호 입력
	- 이메일 입력
	- 연락처 입력
	- 지점명(가게명) 입력
	- 가게 주소 입력
	- 업종 입력(한식,중식,일식.. 등등)<리스트이용
	- 
	
	아이디 중복검사
	1. 사용자가 아이디를 입력하고 중복검사 버튼을 누르면
	2. 버튼 이벤트 발생
	- 입력받은 아이디를 서버로 전송
	3. 서버는 받은 정보를 자신의 파일 시스템(users파일)에서 찾음
	4. 있으면 True 반환, 없으면 False 반환 후 클라이언트로 전송
	5. 클라이언트는 서버로부터 받은 정보로 중복확인
	6. True 면 중복(아이디 사용불가), False 면 중복아님(아이디 사용가능)
	
	이메일 유효성검사
	1. 이메일이 [  XXXXX@XXXX.XXXX  ] 이와 같은 방식으로 입력이 되어있어야 함
	2. 해당 검사는 회원가입 버튼 클릭시 이벤트 발생내용으로 이용

	연락처 유효성검사(선택)
	1. 연락처가 [ 000-0000-0000 ] 이와 같은 방식으로 입력되어있어야 함
	2. 해당 검사는 회원가입 버튼 클릭시 이벤트 발생내용으로 이용
